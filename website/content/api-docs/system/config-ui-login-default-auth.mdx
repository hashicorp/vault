# `/sys/config/ui/login/default-auth`

@include 'alerts/enterprise-only.mdx'

@include 'alerts/restricted-admin.mdx'

Use the `/sys/config/ui/login/default-auth` endpoint to list, create, read,
update, or delete login configurations for the Vault GUI.

Login configurations customize the Vault GUI login form by restricting the
list of authentication methods listed on the login form.

## Create UI default login configuration

This endpoint allows the creation of a new UI default login configuration. Only one UI login configuration is
allowed per namespace. Namespaces can inherit from their parent if the parent's configuration has `disable_inheritance`
set to `false`. If no namespace is provided, the rule will be applied to the root namespace. One of `default_auth_type`,
or `backup_auth_types` is required during creation.

| Method | Path                              |
| :----- | :-------------------------------- |
| `POST` | `/sys/config/ui/login/default-auth/:name` |

### Parameters

- `name` `(string: <required>)` - Path parameter indicating the name of the
  configuration. Names can contain letters, numbers, underscores, and dashes.

- `namespace_path` `(string: "")` - Target namespace for the login configuration.
  Leave `namespace_path` unset to apply the configuration to the `root` namespace.

- `default_auth_type` `(string: <required>)` - The default authentication method.
  You must provide a default method if `backup_auth_types` is unset. Supported
  authentication methods include: `github`, `jwt`, `ldap`, `oidc`, `okta`,
  `radius`, `saml`, `token`, and `userpass`.

- `backup_auth_types` `(array: [])` - A list of backup auth types. Vault
  presents the backup methods in the `Sign in with other methods` tab. You must
  provide at least one backup method if `default_auth_types` is unset. Supported
  authentication methods include: `github`, `jwt`, `ldap`, `oidc`, `okta`,
  `radius`, `saml`, `token`, and `userpass`.

- `disable_inheritance` `(bool: false)` - Defaults to `false` which means child namespaces inherit `default_auth_type` and `backup_auth_types` from their parent. If set to `true`, the config will not be inherited.

### Sample payload

```json
{
  "namespace_path": "it_admins",
  "default_auth_type": "okta",
  "backup_auth_types": ["userpass", "token"],
  "disable_inheritance": "true"
}
```

### Sample request

```shell-session
$ curl --header "X-Vault-Token: ..." \
    --request POST \
    --data @payload.json
    http://127.0.0.1:8200/v1/sys/config/ui/login/default-auth/it-admin-rule
```

## List UI default login configurations

This endpoint returns a list of all UI default login configurations that exist.

| Method | Path                              |
| :----- | :-------------------------------- |
| `LIST` | `/sys/config/ui/login/default-auth` |

### Sample request

```shell-session
$ curl --header "X-Vault-Token: ..." \
    --request LIST \
    http://127.0.0.1:8200/v1/sys/config/ui/login/default-auth
```

### Sample response

```json
{
  "data": {
    "key_info": {
      "default-ldap": {
        "name":"default-ldap",
        "namespace_path":"root",
        "disable_inheritance":false
      }
    },
    "keys":["default-ldap"]
  }
}
```



## Read UI default auth configuration

This endpoint returns the properties of a specific UI default auth configuration.

| Method |                                      |
| :----- | :----------------------------------- |
| `GET`  | `/sys/config/ui/login/default-auth/:name` |

### Parameters

- `name` `(string: <required>)` - The name of the configuration

### Sample request

```shell-session
$ curl --header "X-Vault-Token: ..." \
    http://127.0.0.1:8200/v1/sys/config/ui/login/default-auth/it-admin-rule
```

### Sample response

```json
{
  "data": {
    "backup_auth_types":["token","userpass"],
    "default_auth_type":"okta",
    "disable_inheritance":false,
    "namespace_path":"it_admins"
  }
}
```

## Update UI default auth configuration

This endpoint updates the properties of a specific UI default auth configuration.

| Method |                                      |
| :----- | :----------------------------------- |
| `POST` | `/sys/config/ui/login/default-auth/:name` |

### Parameters

- `namespace_path` `(string: "")` - Namespace to apply this configuration to.

- `default_auth_type` `(string: <required>)` - The type of auth method to be presented by default. Only UI supported auth types are allowed: `github`,`jwt`,`ldap`,`oidc`,`okta`,`radius`,`saml`,`token`,and `userpass`

- `backup_auth_types` `(array: [])` - Backup auth types to be presented in the `Sign in with other methods` tab. Only UI supported auth types are allowed: `github`,`jwt`,`ldap`,`oidc`,`okta`,`radius`,`saml`, `token`,and `userpass`

- `disable_inheritance` `(bool: false)` - Disable child namespaces from inheriting `default_auth_type` and `backup_auth_types` from this configuration.

### Sample payload

```json
{
  "namespace_path": "it_admins",
  "default_auth_type": "ldap",
  "backup_auth_types": ["userpass", "github"],
  "disable_inheritance": "true"
}
```

### Sample request

```shell-session
$ curl --header "X-Vault-Token: ..." \
    --request POST \
    --data @payload.json
    http://127.0.0.1:8200/v1/sys/config/ui/login/default-auth/it-admin-rule
```
