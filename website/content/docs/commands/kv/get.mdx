# `kv get`

Read data from a key/value plugin.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault kv get [flags] <secret_path>

$ vault kv get [-help | -h]
```

</CodeBlockConfig>



## Description

`kv get` returns values stored at the provided secret path for the targeted `kv`
plugin. Key/value v2 plugins also return metadata associated with the secret
path.

<Tip title="Related API endpoints">

- ReadSecret (v1) - [`GET: /secret/:path`](/vault/api-docs/secret/kv/kv-v1#read-secret)
- ReadSecretVersion (v2) - [`GET: /:secret-mount-path/data/:path?version=:version-number`](/vault/api-docs/secret/kv/kv-v2#read-secret-version)

</Tip>

## Examples

Read metadata and all the current key/value pairs on the path `creds` from the
`kv` v2 plugin mounted at `shared/`:

```shell-session
$ vault kv get -mount shared creds

== Secret Path ==
shared/data/creds

======= Metadata =======
Key                Value
---                -----
created_time       2022-06-15T20:23:40.067093Z
custom_metadata    <nil>
deletion_time      n/a
destroyed          false
version            1

====== Data ======
Key         Value
---         -----
passcode    my-long-passcode
```

Read the current key/value pairs on the path `creds` from the `kv` v2 plugin
mounted at `shared/`:

```shell-session
$ vault kv get -mount shared creds

====== Data ======
Key         Value
---         -----
passcode    my-long-passcode
```

Limit `kv get` output from the `creds` path to the `passcode` entry:

```shell-session
$ vault kv get -mount shared -field "passcode" creds

my-long-passcode
```
