# `kv list`

List keys for the given secret path in a key/value plugin.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault kv list [flags] <secret_path>

$ vault kv list [-help | -h]
```

</CodeBlockConfig>



## Description

`kv list` returns all key names available at the provided secret path. Sub-paths
include a `/` suffice. **Do not encode sensitive information in key names**.
While `kv list` does not return value data, Vault does not apply policy-based
filtered to the results. 

<Tip title="Related API endpoints">

  - ListSecrets (v1) - [`LIST: /secret/:path`](/vault/api-docs/secret/kv/kv-v1#list-secrets)
  - ListSecrets (v2) - [`LIST: /:secret-mount-path/metadata/:path`](/vault/api-docs/secret/kv/kv-v2#list-secrets)

</Tip>

## Examples

List keys for the key/value pairs on the path `my-app` from the `kv` v2 plugin
mounted at `shared/`:

```shell-session
$ vault kv list -mount shared my-app/

Keys
----
admin_creds
domain
eng_creds
qa_creds
release
```
