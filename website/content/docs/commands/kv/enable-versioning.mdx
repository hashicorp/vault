# kv enable-versioning

Turns on versioning for the backend at the provided path.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault kv enable-versioning [flags] <key>

$ vault kv enable-versioning [-help | -h]
```

</CodeBlockConfig>

## Description

`kv enable-versioning` turns on versioning for an existing non-versioned 
key/value secrets engine (KV version 1) at its path.

The Vault CLI follows consistent exit code conventions to indicate different
