# secrets enable

Enables a secrets engine.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault secrets enable [flags] <type>

$ vault secrets enable [-help | -h]
```

</CodeBlockConfig>

`secret enable` activates a secrets engine at your chosen path. Vault will 
return an error if another secrets engine already exists at that path. By 
default, Vault enables secrets engines at a path matching their `type`, but 
you can specify a custom path using the `-path` option.

After activation, you will need to configure your secrets engine based on its 
specific requirements. Refer to the [secrets engine documentation](/vault/docs/secrets) for specific 
configuration options.

## Examples

Enable a secrets engine:

```shell-session
$ vault secrets enable database
Success! Enabled the database secrets engine at: database/
```

Enable the AWS secrets engine at `aws/`:

```shell-session
$ vault secrets enable aws
Success! Enabled the aws secrets engine at: aws/
```

Enable the SSH secrets engine at `ssh-prod/`:

```shell-session
$ vault secrets enable -path=ssh-prod ssh
```

Enable the database secrets engine with an explicit maximum TTL of 30m:

```shell-session
$ vault secrets enable -max-lease-ttl=30m database
```

Enable a custom plugin (after it is registered in the plugin registry):

```shell-session
$ vault secrets enable -path=my-secrets my-plugin
```
