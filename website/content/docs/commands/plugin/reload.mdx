# plugin reload

Reloads mounted plugin backends.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault plugin reload [flags]

$ vault plugin reload [-help | -h]
```

</CodeBlockConfig>

## Description

`plugin reload` reloads mounted plugin backends. You must specify either the 
plugin name (`plugin`) or the desired plugin backend mounts (`mounts`), but not 
both. If you provide the plugin name, all mounted paths that use that plugin 
backend will be reloaded.

If you run this command with a Vault namespace other than the root namespace, 
only plugins running in the same namespace will be reloaded.

<Tip title="Related API endpoints">

- ReloadPlugin - [`POST: /sys/plugins/reload`](/vault/api-docs/system/plugins-reload#reload-plugin)
- ReloadPluginNamespace - [`POST: /sys/plugins/reload/backend`](/vault/api-docs/system/plugins-reload#reload-plugins-within-a-namespace)

</Tip>

## Examples

Reload a plugin by type and name:

```shell-session
$ vault plugin reload -type=auth -plugin my-custom-plugin
Success! Reloaded plugin: my-custom-plugin
```

Reload an auth plugin by mount:

```shell-session
$ vault plugin reload \
  -mounts auth/my-custom-plugin-1 \
  -mounts auth/my-custom-plugin-2
Success! Reloaded mounts: [auth/my-custom-plugin-1/ auth/my-custom-plugin-2/]
```

Reload a secrets plugin by mount:

```shell-session
$ vault plugin reload \
  -mounts my-custom-plugin-1 \
  -mounts my-custom-plugin-2
Success! Reloaded mounts: [my-custom-plugin-1/ my-custom-plugin-2/]
```

Reload a secrets plugin named "my-custom-plugin" across all nodes and replicated clusters:

```shell-session
$ vault plugin reload \
  -type=secret \
  -plugin=my-custom-plugin \
  -scope=global
```
