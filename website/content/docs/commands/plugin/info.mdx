# plugin info

Displays information about a plugin in the catalog with the given type and name.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault plugin info [flags] <type> <name>

$ vault plugin info [-help | -h]
```

</CodeBlockConfig>

## Description

The `plugin info` displays information about a plugin in the catalog.
You must include the plugin's type of `"auth"`, `"database"`, or `"secret"` in 
the command.

<Tip title="Related API endpoints">

ReadPlugin - [`GET: /sys/plugins/catalog/:type/:name?version=:version`](/vault/api-docs/system/plugins-catalog#read-plugin)

</Tip>

### deprecation_status field

As of 1.12, all builtin plugins will have an associated Deprecation
Status. This status will be reflected in the `deprecation_status` key/value
pair.

## Examples

Display information about an auth plugin:

```shell-session
$ vault plugin info -version=v1.0.0 auth my-custom-plugin

Key                   Value
---                   -----
args                  []
builtin               false
command               my-custom-plugin
deprecation_status    n/a
name                  my-custom-plugin
oci_image             n/a
runtime               n/a
sha256                04ce575260fa3a2cfc477d13ac327108c50838a03917ec4d6df38ecdc64452d1
version               v1.0.0
```

Display information about a database plugin:

```shell-session
$ vault plugin info database postgresql-database-plugin
Key                   Value
---                   -----
args                  []
builtin               true
command               n/a
deprecation_status    supported
name                  postgresql-database-plugin
oci_image             n/a
runtime               n/a
sha256                n/a
version               n/a
```
